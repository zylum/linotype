# Linotype v0.6 and v0.6.1

v0.6 adds **LinoLoop**, a thin wrapper over executor briefs that can run a single galley or an ordered release list.

v0.6.1 adds worktree-aware execution modes and safety controls.

## What changed

### LinoLoop command

```bash
cli/linoloop <galley-name>
cli/linoloop <release-id>
```

Behavior:
- Generates executor brief via `cli/linotype exec brief <galley>`
- Supports isolated mode with one branch/worktree per galley
- Uses a loop runner (default `ralph`) when installed
- Falls back to printing the brief for manual paste if no runner is available

Compatibility alias:
- `cli/linotype exec opencode <galley>`

### Modes (v0.6.1)

- `auto` (default): single galley -> `direct`, release -> `serial-isolated`
- `direct`: run in current working tree
- `serial-isolated`: one worktree/branch per galley

Common flags:
- `--worktree-root <path>`
- `--reuse-worktree`
- `--require-clean-git` (default) / `--allow-dirty`
- `--dry-run`
- `--auto-pr` (placeholder; records manual follow-up)

### Release format

Release file:
- `docs/work/releases/<release-id>/galleys.txt`

Rules:
- one galley ID per line
- blank lines allowed
- comment lines start with `#`

### Environment

- `LINOLOOP_AGENT` (default `opencode`)
- `LINOLOOP_MAX_ITER` (default `50`)
- `LINOLOOP_RUNNER` (default `ralph`)
- `LINOLOOP_MODE` (default `auto`)
- `LINOLOOP_WORKTREE_ROOT` (default `dist/linoloop/worktrees`)

Release status log:
- LinoLoop appends release timeline events to `docs/work/releases/<release-id>/status.md`.
